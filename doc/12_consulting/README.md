# Consulting

Consulting describes how Protopipe becomes useful
inside an existing organization.

This layer contains canonical consulting concepts:
- observable entry points
- engagement behavior
- mandate expansion logic

Consulting concepts are not mechanics.
They do not enforce structural consequences.

Consulting concepts are not experiences.
They are not narratives or examples.

They define stable language for how Protopipe enters,
earns trust, and expands through evidence.

---

## Boundary

Consulting documents may define:
- entry conditions
- consulting behavior
- evidence requirements
- failure modes of engagement

They must not define:
- customer-specific decisions
- sales scripts
- implementation plans
- staffing models
- transformation programs

Consulting concepts are valid
only when they can be traced back to
existing Noesis concepts
or recurring Protopipe presentation language.
